# 27 задача(программируемая для досрока)
## a:
1) загрузить данные с файла и удалить первый элемент
2) создать переменную по длине списка
3) нужно удвоить список, чтобы на его основе создавать рабочие срезы
4) в цикле мы создаем список на основе среза сдвоеного списка(при каждом новом срезе нужно обнулять стоимость)
5) испольуя созданный список считаем стоимость для первого элемента по формуле
6) смещая срез находим стоимости относительно всех пунктов(важно, что при кольцевой дороге индексы пересчитываются на убывание после середины, для этого стоит создать еще одну переменную)
7) стоимости пунктов накапливаем в списке и после берем индекс минимальной из них + 1(поправка к нумерации)
## b:
актуально все то, что в а, но цикл перебора нужно организовывать по диапазону, в аргумента хкоторого есть старт финиш и шаг
используется замер выборочных километров (равно распределенных)
организуем глобальный цикл, где старт финиш и шаг будут изменяться
после каждого глобального прохода уменьшаем границы диапазона, уменьшая их
условие выхода из глобального цикла - повторение ответа при шаге 1
если шаг = 0, то шаг устанавливается 1
целесообазно на каждом проходе писать новые значения старта, финиша и шага
границы диапазона старт и финиш определяются по минимальному километражу +- шаг
после пересчета старта и финиши пересчитываем старт, целесообразно брать 20 замеров на диапазон
